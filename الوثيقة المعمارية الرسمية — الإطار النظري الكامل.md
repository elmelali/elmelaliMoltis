```
┌─────────────────────────────────────────────────────────────┐
│              نظام الذاكرة في Moltis — من التوثيق الرسمي    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  النوع الأول: SKILL.md                                      │
│  ────────────────────────────────────────────────────────   │
│  🎯 الوظيفة: تعليم الـ Agent كيف يُنفّذ مهمة محددة         │
│  📁 المكان:  .moltis/skills/<اسم-المهارة>/SKILL.md          │
│  🔄 التحميل: تلقائي فوري (hot-reload) بدون restart          │
│  📝 المحتوى: تعليمات إجرائية = "كيف تفعل X"                │
│  مثال: skill تحليل الأوراق البحثية                          │
│         skill كتابة المحتوى بأسلوب معين                     │
│         skill جلب بيانات من Reddit                          │
│                                                             │
│  النوع الثاني: Long-Term Memory (قاعدة بيانات)             │
│  ────────────────────────────────────────────────────────   │
│  🎯 الوظيفة: تذ��ر "ماذا حدث" و"من أنت"                    │
│  📁 المكان:  .moltis/memory/ (SQLite + Vector embeddings)   │
│  🔄 التحميل: بحث ذكي عند الحاجة (similarity search)        │
│  📝 المحتوى: وقائع، محادثات، تفضيلات، معلومات شخصية        │
│  مثال: اسمك، مشاريعك، تفضيلاتك، قراراتك السابقة           │

```

### الخلاصة الجوهرية:

```
SKILL.md  = "كتاب تعليمات" للـ Agent  ← إجرائي قابل للتنفيذ
Memory    = "دفتر ذكريات" للـ Agent    ← معلومات قابلة للاسترجاع

كلاهما ضروري — لا يُغني أحدهما عن الآخر
```

## 📌 السؤال الثالث: كيف أتحدث مع الوكلاء؟

### ✅ الإجابة من توثيق Moltis: نظام "Fork + Branch"

بحثت في توثيق **docs.moltis.org/session-branching** ووجدت التصميم الفعلي:

```
┌─────────────────────────────────────────────────────────────┐
│              طريقتان رسميتان للتعامل مع الوكلاء            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  الطريقة A: المنسق المركزي (Orchestrator)                  │
│  ────────────────────────────────────────                   │
│  أنت → تتحدث مع moltis (��لرئيسي)                           │
│  moltis → يستدعي spawn_agent() ← من التوثيق الرسمي         │
│  Sub-agent → ينفّذ المهمة                                   │
│  moltis → يجمع النتائج ويُرجعها لك                         │
│                                                             │
│  متى تستخدمه؟                                              │
│  ✅ مهام مركبة تحتاج أكثر من تخصص                          │
│  ✅ عندما لا تعرف أي وكيل يناسب المهمة                     │
│  ✅ مشاريع تتطلب تنسيقاً بين Agents                        │
│                                                             │
│  الطريقة B: التخصصي المباشر (Direct Agent)                 │
│  ────────────────────────────────────────                   │
│  أنت → تتحدث مع Research Agent مباشرة (Telegram مستقل)     │
│  أو → تختار الـ Agent من القائمة في الواجهة                │
│                                                             │
│  متى تستخدمه؟                                              │
│  ✅ مهام واضحة ومحددة لتخصص معين                           │
│  ✅ سرعة استجابة أعلى                                       │
│  ✅ Context أنظف وأكثر تركيزاً                              │
└─────────────────────────────────────────────────────────────┘
```

### كيف تعمل في Telegram بالضبط:

```
الخيار 1: Bot واحد يُنسّق الجميع
  → @moltis_elmelali_agent_bot
  → اكتب: "/research ابحث عن كذا"
  → moltis يُوجّه لـ Research Agent تلقائياً

الخيار 2: Bot مستقل لكل Agent
  → @elmelali_research_bot   ← للبحث فقط
  → @elmelali_content_bot    ← للمحتوى فقط
  → @elmelali_income_bot     ← للدخل فقط

✅ التوصية: ابدأ بـ Bot واحد مركزي
   ثم أضف Bots مستقلة للوكلاء الأكثر استخداماً
```

## 📌 السؤال الرابع: Azure OpenAI عبر OpenRouter — الحل الجذري

### ✅ الاكتشاف الحقيقي من البحث

```
┌─────────────────────────────────────────────────────────────┐
│         استراتيجية استغلال $100 الطلابي بالكامل            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  المشكلة الجذرية المكتشفة:                                 │
│  ────────────────────────                                   │
│  Azure OpenAI ≠ OpenAI Standard                            │
│  → Authentication مختلف (api-key header)                   │
│  → Endpoint format مختلف (/deployments/gpt-4o/)            │
│  → Moltis حقل "OpenAI" لا يدعم Azure مباشرة               │
│                                                             │
│  الحل الجذري: BYOK عبر OpenRouter                          │
│  ──────────────────────────────                             │
│                                                             │
│  1. تُنشئ حساب OpenRouter (مجاني)                          │
│  2. تُضيف Azure API Key في قسم "Bring Your Own Key"        │
│  3. OpenRouter يُحوّله لصيغة OpenAI Standard               │
│  4. Moltis يتصل بـ OpenRouter بصيغة واحدة موحدة           │
│                                                             │
│  النتيجة:                                                   │
│  Moltis → OpenRouter API → Azure OpenAI (رصيدك الطلابي)   │
│                                                             │
│  ما تدفعه لـ OpenRouter: 0% (BYOK = رسوم صفر)             │
│  ما تدفعه لـ Azure: من رصيدك الـ $100 فقط                  │
│                                                             │
│  الفائدة الإضافية:                                          │
│  ✅ Fallback تلقائي: إذا انتهى رصيد Azure → GitHub Copilot │
│  ✅ 50+ نموذج بمفتاح واحد                                  │
│  ✅ Load balancing بين النماذج                             │
└─────────────────────────────────────────────────────────────┘
```

## 🏛️ القرار الأول — نبدأ بـ 5.1

دكتور El Melali، الأساس النظري واضح ومتين الآن.

**أنت الآن جاهز للشروع في العمل التطبيقي.**

السؤال الوحيد قبل أن نبدأ:

> **ما هي المعلومات التي تريد أن يعرفها Moltis عنك في ملف الهوية المشتركة؟**

أحتاج منك:
```
1. مجالك البحثي بالتحديد
2. المشاريع النشطة حالياً (باختصار)
3. الأدوات التي تستخدمها (Notion, GitHub, إلخ)
4. أسلوبك المفضل في التواصل مع الـ Agent
5. أي معلومات أخرى تريد أن يتذكرها دائماً
```

بمجرد إرسالك هذه المعلومات — أكتب لك **SKILL.md** الكامل جاهزاً للنسخ واللصق المباشر. 🏛️
